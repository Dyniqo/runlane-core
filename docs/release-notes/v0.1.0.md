# Runlane Core v0.1.0 Release Notes

Runlane Core v0.1.0 delivers the backend core for workflow automation, webhook ingestion, queue-backed execution, API integrations, AI decision steps, Stripe billing synchronization, usage metering and deployment validation.

## Highlights

- Modular DDD monorepo with independent API and Worker runtimes
- Workspace-scoped authentication, sessions, API keys and object-level authorization
- Workflow definitions with versioning, publish lifecycle and public IDs
- Signed webhook ingestion with timestamp validation, replay protection and idempotency
- API-key protected automation bridge for external automation callers
- BullMQ execution queue with workspace-scoped job payloads
- Worker-side execution engine with sequential steps, retry rules, dead-letter state and manual retry
- Encrypted workflow secrets and connector credentials
- HTTP connector with timeout, redirect limits, response-size limits, DNS validation and SSRF protection
- OpenAI-compatible AI provider adapter and AI decision step
- Slack and Discord notification connectors
- Usage tracking, plan enforcement and audit logging
- Stripe checkout, customer portal and webhook subscription synchronization
- Demo seed, reset and session-scoped demo workspace isolation
- Docker Compose runtime and image-based deployment stack with Caddy
- GitHub Actions verification, GHCR image publishing and deployment image smoke workflow
- Postman collection, demo scripts, validation scripts and operational documentation

## Validation commands

Run the release gate locally:

```powershell
pnpm format
pnpm verify
```

Run API-level validation with API and Worker running:

```powershell
pnpm validate:integration
pnpm validate:demo
pnpm validate:demo-isolation
pnpm validate:http-connector
```

Run Docker clean-room validation when Docker or deployment files change:

```powershell
pnpm validate:clean-room
```

## Deployment image flow

Release images are published with commit SHA tags:

```txt
ghcr.io/dyniqo/runlane-core-api:sha-<commit-sha>
ghcr.io/dyniqo/runlane-core-worker:sha-<commit-sha>
ghcr.io/dyniqo/runlane-core-migrator:sha-<commit-sha>
```

Use the Deployment image smoke workflow to verify published images against the image-based Compose stack.
